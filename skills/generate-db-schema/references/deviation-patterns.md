# 乖離（retrenchment）パターン集 (v2.1.0)

DDL が VDM-SL 仕様を忠実に表現できない箇所の典型パターンと、補償手段の割り当て方。
DEVIATIONS.md の各行は必ずこのカタログの ID 系列（またはプロジェクト固有 ID）を使う。

## 原則

- **乖離は失敗ではなく設計情報である**。real を float に写せないのは実装の宿命であり、
  隠すことだけが失敗（retrenchment の考え方：Banach & Poppleton）
- 補償手段は必ず実在する機構を指す：**DB トリガー / アプリ層契約チェック（generate-code
  の checkPre・checkPost・checkInv）/ 契約テスト（generate-tests）** のいずれか
- 乖離の密度が高い場所ほどバグの温床 — generate-tests は DEVIATIONS.md を読んで
  乖離箇所の境界値テストを優先生成する（将来連携）

## カタログ

### D-NUM: 数値の有界化

| 項目 | 内容 |
|---|---|
| 乖離 | `nat`/`nat1`/`int`（無限）→ BIGINT（±2^63）/ INTEGER |
| リスク | オーバーフロー。特に集計・乗算 |
| 補償 | 実用上到達不能ならその旨を記録。到達しうる（カウンタ・金額集計）なら アプリ層チェック + 境界値テスト |

### D-FLOAT: 実数の浮動小数点化

| 項目 | 内容 |
|---|---|
| 乖離 | `real`（実数）→ DOUBLE PRECISION（IEEE754） |
| リスク | 丸め誤差。等値比較を含む不変条件は DB 上で偽になり得る |
| 補償 | 金額・比率など精度が意味を持つ場合は NUMERIC(p,s) へ変更（要ユーザー確認）。 等値を含む inv は誤差許容比較のアプリ層チェックに変換 |

### D-STR: 文字列の表現差

| 項目 | 内容 |
|---|---|
| 乖離 | `seq of char` → VARCHAR/TEXT。照合順序・Unicode 正規化・空白の扱い |
| リスク | `len` と CHAR_LENGTH の不一致（結合文字）、大文字小文字を区別しない比較 |
| 補償 | 一意性が正規化に依存する場合は正規化をアプリ層で強制し、その関数を明記 |

### D-TRANS: 遷移規則の CHECK 不可能性

| 項目 | 内容 |
|---|---|
| 乖離 | 事前条件の新旧比較（`ValidTransition(old, new)`）は CHECK で書けない |
| 補償 | 第一候補: UPDATE トリガー（R12）。トリガーを採らない場合はアプリ層 + 遷移全パターンの契約テストを**必須**とする |

### D-CROSS: 複数行・複数テーブル不変条件

| 項目 | 内容 |
|---|---|
| 乖離 | `forall`/集約を含む状態不変条件は単一行 CHECK で書けない |
| 補償 | 構造で消す（SEQUENCE 化等）> トリガー > 遅延制約 > アプリ層+テスト の優先順で選び、 選ばなかった選択肢と理由を一行残す |

### D-SEQ: 採番の SEQUENCE 置換

| 項目 | 内容 |
|---|---|
| 乖離 | `nextId` 型のカウンタを DB SEQUENCE/IDENTITY に置換すると、ロールバックで欠番が 生じ、`post RESULT = nextId~` のような「隙間なし」を含意する契約と食い違い得る |
| 補償 | 仕様側の契約が欠番を許容するか確認。許容しないなら SEQUENCE を使わず シングルトン状態テーブル + SERIALIZABLE（性能コストをコメントに明記） |

### D-PUNIQ: 条件付き一意性

| 項目 | 内容 |
|---|---|
| 乖離 | 「〜の中でのみ一意」は標準 UNIQUE で書けない（PG は部分インデックスで可） |
| 補償 | PG: 部分 UNIQUE インデックス。MY/LT: トリガーまたはアプリ層+テスト |

### D-ARR: 配列カラムの採用

| 項目 | 内容 |
|---|---|
| 乖離 | `set of` 基本型を PG 配列にすると、要素の型不変条件・重複禁止が DDL で書きにくい |
| 補償 | 重複禁止が仕様にあるなら結合テーブルに戻す。配列を採る場合は要素検査を アプリ層チェックに割り当てる |

### D-ENUM: 列挙型の進化

| 項目 | 内容 |
|---|---|
| 乖離 | quote 型の値は仕様上自由に増減できるが、PG の ENUM は追加のみ（`ALTER TYPE ... ADD VALUE`）で削除・並べ替えは不可 |
| 補償 | 仕様の quote 値変更時は DDL を再導出しマイグレーションを発行。変更が頻繁なら TEXT + CHECK を選択 |

### D-TX: 分離レベルと並行性

| 項目 | 内容 |
|---|---|
| 乖離 | VDM の操作は逐次意味論。既定の READ COMMITTED では、他行の不在や集約値に 依存する事後条件が並行実行で破れ得る（write skew 等） |
| 補償 | 該当する操作を列挙し、分離レベル引き上げ or 一意制約/トリガーによる防御を 選択。選択と理由を schema.sql のトランザクション対応節に記す |

### D-NULL: NULL の三値論理

| 項目 | 内容 |
|---|---|
| 乖離 | VDM の `nil` と SQL の NULL は比較意味論が異なる（NULL = NULL は UNKNOWN）。 UNIQUE は NULL を重複とみなさない（PG 15 の NULLS NOT DISTINCT を除く） |
| 補償 | optional フィールドを含む一意性・等値の不変条件は、意図（nil 同士は等しいか）を ユーザーに確認してから写像を決める |
