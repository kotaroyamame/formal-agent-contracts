EDGE CASES HANDLED ACROSS ALL RUNS
================================================================================

1. Negative Initial Balance
   - Rejected via precondition
   - Catches: Invalid account creation attempts

2. Zero-Amount Operations
   - Zero deposits: Rejected (must be positive)
   - Zero withdrawals: Rejected (must be positive)
   - Zero transfers: Rejected (must be positive)
   - Zero initial balance: Allowed (valid state)

3. Whitespace-Only Names
   - Explicitly trimmed before validation
   - Whitespace-only after trim: Rejected
   - Names with internal spaces: Accepted
   - Empty names: Always rejected

4. Transfer Between Same Account
   - Precondition: fromId ≠ toId
   - Always caught and rejected

5. Withdrawal Limit Enforcement
   - Global limit: 1,000,000 per operation
   - Applies to both withdrawals and transfers
   - Exact boundary testing: 1,000,000 allowed, 1,000,001 rejected

6. Balance Never Negative
   - Preconditions guard against negative balance
   - Postconditions verify non-negative result
   - Invariants ensure state consistency

7. Transfer Atomicity
   - Both accounts updated or neither updated
   - Money conservation guaranteed: total before = total after
   - Rollback on any error (Runs 3 & 5)

8. Account Isolation
   - Operations on one account don't affect others
   - Postcondition verification: other accounts unchanged

9. Maximum Balance
   - No deposit creates balance ≥ 1,000,000,000
   - Precondition check prevents overflow
   - Edge case: 999,999,998 + 1 allowed, +2 rejected

10. Exact Boundary Values
    - Balance = withdrawal amount: Allowed (to exactly zero)
    - Amount = limit (1M): Allowed
    - Amount = limit + 1: Rejected
    - All tested in each run
