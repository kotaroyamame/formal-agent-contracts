12 EDGE CASE TRAPS
================================================================================

1. Reservation System & Book Availability States
   - Tests catalog's available_copies tracking
   - Catches: stock going to zero

2. Return-Then-Immediately-Borrow (Overdue Flag Clearing)
   - Tests overde flag management across returns
   - Catches: flag cleared incorrectly when other loans overdue

3. 5-Book Count Boundary
   - Tests max loan limit (5)
   - Catches: allowing 6th loan, boundary off-by-one

4. Stock Consistency Between Agents
   - Tests that available + reserved = total
   - Catches: invariant violations in stock updates

5. Double Extension Attempt
   - Tests preventing extension of already-extended loan
   - Catches: precondition check failures

6. Extended-Then-Overdue Handling
   - Tests temporal ordering after extension
   - Catches: due_date <= borrow_date violations

7. Multiple Copies of Same Book
   - Tests separate tracking of multiple copies
   - Catches: shared loan state across copies

8. Last Copy Reservation Conflict
   - Tests borrowing when last copy reserved
   - Catches: zero-copy scenarios

9. Duplicate Member Registration
   - Tests preventing same memberId twice
   - Catches: no unique constraint checking

10. Invalid Return of Non-Borrowed Book
    - Tests preconditions on returnLoan
    - Catches: returning loans not in active set

11. Precious Book Lending Prohibition
    - Tests that precious books are never loaned
    - Catches: precious category enforcement

12. Day 14 Boundary (Return on Day 14 is NOT Overdue)
    - Tests IsOverdue logic: due_date < current_date (not <=)
    - Catches: off-by-one in overdue date calculations

================================================================================
BUSINESS RULES IMPLEMENTED
================================================================================

All 5 runs enforce these invariants:

Member Rules:
- Maximum 5 concurrent loans per member (loan_count <= 5)
- Cannot borrow while has_overdue = true
- No duplicate member registrations

Book Rules:
- Available copies never exceed total copies (available_copies <= total_copies)
- Precious books (category=PRECIOUS) cannot be loaned
- Cannot borrow when available_copies = 0

Loan Rules:
- Loan period: 14 days (borrow_date + 14 = due_date)
- Extension: +7 days (due_date + 7 after first extension)
- Extension allowed: once per loan only (extended=false precondition)
- Cannot extend returned loans (is_returned=false precondition)
- Cannot return twice (is_returned=false precondition)

Temporal Rules:
- Due date always > borrow date (due_date > borrow_date)
- Overdue check: due_date < current_date AND not is_returned
- Day 15 (when due_date=15) is NOT overdue if current_date=15

================================================================================
AGENTS IMPLEMENTED
================================================================================

CATALOG AGENT:
- Manages book catalog and inventory
- Operations: AddBook, ReserveBook, ReturnBook
- Reads: GetAvailableCopies, IsPrecious, GetBookDetails
- Invariant: Book.available_copies <= Book.total_copies

MEMBER AGENT:
- Manages patron registration and loan status
- Operations: RegisterMember, IncrementLoanCount, DecrementLoanCount, SetOverdue, ClearOverdue
- Reads: CanBorrow, GetLoanCount, HasOverdue
- Invariant: Member.loan_count in [0, 5]

LOAN AGENT:
- Manages loan lifecycle (create, extend, return)
- Operations: CreateLoan, ExtendLoan, ReturnLoan
- Reads: IsOverdue, GetMemberLoans, GetActiveLoanCount, GetLoan
- Invariant: Loan.due_date > Loan.borrow_date

ORCHESTRATOR (LibrarySystem):
- Coordinates multi-agent workflows
- High-level operations: borrowBook, returnBookWorkflow, extendLoan
- Ensures pre/post conditions across agent boundaries
- Handles state consistency across operations
