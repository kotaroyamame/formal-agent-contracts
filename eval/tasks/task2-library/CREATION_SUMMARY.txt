================================================================================
KEY DESIGN DECISIONS
================================================================================

1. Agent Responsibility Distribution:
   - CatalogAgent: Book catalog + inventory (stock management)
   - MemberAgent: Member registry + loan count + overdue status
   - LoanAgent: Loan records + due dates + overdue calculation

2. Loan Count Tracking (T2-03):
   - Managed by MemberAgent, not derived from LoanAgent
   - Incremented on successful borrow
   - Decremented on return
   - Checked against 5-book limit in is_eligible_for_loan

3. Overdue Status Management (T2-02):
   - set_overdue(member) when any loan becomes overdue
   - clear_overdue(member) when ALL active loans are on-time or returned
   - Boolean flag, not count of overdue loans

4. Extension Constraints (T2-05):
   - is_extended field on Loan (boolean)
   - Pre-condition: is_extended = false before extension
   - Once extended, is_extended = true, no further extensions

5. Boundary Handling (T2-12):
   - Day 14 is the due date (inclusive)
   - Return on day 14 = on-time
   - Return on day 15+ = overdue
   - Comparison: return_date > due_date

6. Stock Management (T2-04, T2-07, T2-08):
   - Per-book stock maintained in CatalogAgent
   - Decreased on successful borrow
   - Increased on return
   - Boundary: stock must be > 0 to borrow

7. Precious Book Handling (T2-11):
   - Category = <Precious> cannot be loaned
   - Checked in can_be_loaned predicate
   - Rejection at borrow time

8. Invalid Operations (T2-10):
   - Return requires valid active loan
   - find_active_loan returns nil if not found
   - Operation fails with clear error message
