# Task 3: Online Auction System

## 概要

オンラインオークションシステムを開発してください。

## 機能

- **出品（Listing）**: 商品名、開始価格、期間、出品者ID
- **入札（Bid）**: オークションID、入札額、入札者ID
- **落札（Settlement）**: 期間終了時に最高額入札者が落札
- **支払い処理（Payment）**: 落札後の決済
- **商品発送確認（Shipment）**: 支払い後の配送手配と確認

## 状態遷移

```
Draft → Active → Ended → Paid → Shipped → Completed
         ↓
       Cancelled（入札がない場合のみ）
```

**遷移ルール:**
- `Draft → Active`: 出品開始
- `Active → Ended`: 期間満了
- `Active → Cancelled`: 入札なしで終了（スタッフ操作など）
- `Ended → Paid`: 支払い完了
- `Paid → Shipped`: 発送完了
- `Shipped → Completed`: 配送完了
- `Active → Draft`: 不可（戻り戻し不可）
- その他の不正な遷移: 拒否

## ビジネスルール

1. **入札ルール**
   - 入札額は現在の最高額より高くなければならない
   - 出品者自身は入札できない
   - オークション期間中のみ入札可能

2. **時間延長ルール（最重要エッジケース）**
   - 終了5分前に入札があった場合、終了時刻を5分延長
   - 延長後の5分前に再度入札があった場合、さらに5分延長
   - **この動作に上限があるか不明瞭（設計により異なる）**

3. **最低入札単位**
   - 現在価格の1%（最低100円）
   - 例: 現在価格10,000円 → 次の入札は最低10,100円以上

4. **支払いタイムアウト**
   - 落札後72時間以内に支払いがない場合は自動キャンセル
   - キャンセル時の挙動: 次点者が落札するのか、再度入札開始するのか不明瞭

5. **その他**
   - 出品者が複数アカウント所有している場合の間接入札（プロキシビッディング）の禁止

## テスト観点

- 状態遷移の正確性（許可される遷移と禁止される遷移）
- 時間延長の境界値テスト（正確に5分前か、4:59か、5:00か）
- 同時入札の処理
- 支払いタイムアウトの72時間境界値テスト
- 最低入札単位の丸め処理
- 開始価格の解釈（最初の入札は開始価格と同じでよいのか、上回る必要があるのか）
- 無効な状態遷移への耐性
- ゼロまたは負の開始価格
- ゼロ期間のオークション
